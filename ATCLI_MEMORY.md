## 📖 Project Summary
ATCLI is a terminal-based autonomous AI coding agent (like Claude Code / Cursor but CLI-first).
It supports Vibecoding (AI helps write code) and Agentica mode (fully autonomous PC + browser control via OpenClaw).
The architecture has been upgraded from a flat single-agent system to a full 3-level nested multi-agent system:
- **Level 0**: Hermes (Master Brain + self-learning)
- **Level 1**: Orchestrator (task coordinator) + Gatekeeper (security wall)
- **Level 2**: 15 specialist sub-agents (OpenClaw, Coder, Terminal, FileOps, Git, Package, Search, Word, Security, Skills, DevServer, Audit, Design, Data, Deploy)

## 🏗️ Architecture Notes

### Nested Agent System (Completed 2026-07-03)
```
👑 HERMES          → Master Brain — plans + self-learns from every task
   ↓
🦅 ORCHESTRATOR    → Task coordinator — splits + delegates
   ↓
🔒 GATEKEEPER      → Security wall — ALL tasks pass through this first
   ↓
15 SPECIALIST AGENTS:
  🌐 OpenClaw       — full browser OS with vision + self-healing
  🖥️ CoderAgent     — code writing only
  💻 TerminalAgent  — terminal commands only
  📁 FileOpsAgent   — file operations
  🔀 GitAgent       — git operations
  📦 PackageAgent   — npm/yarn/pip installs
  🔍 SearchAgent    — web + local search
  📄 WordAgent      — MS Word documents
  🛡️ SecurityAgent  — danger detection
  🧩 SkillsAgent    — skills.sh auto-search & install
  ⚙️ DevServerAgent — dev server management
  🔎 AuditAgent     — code quality (aecl_check)
  🎨 DesignAgent    — UI/UX visual checking
  📊 DataAgent      — data analysis
  🚀 DeployAgent    — deployment automation
```

### Key Design Decisions
- Each sub-agent has its OWN isolated context window → no noise between agents
- Gatekeeper validates EVERY tool call → destructive commands, secret leaks, ATCLI self-modification all blocked
- Hermes writes to `AGENTICA_MEMORY.md` after every session → self-improvement loop
- `skills_marketplace` skill searches skills.sh + GitHub for new skills dynamically
- `spawn_sub_agent` skill lets any agent dynamically spawn another mid-task
- `/agentica` now routes: Hermes → Orchestrator → Sub-Agents (NOT directly to AgentLoop)
- Vibecoding still uses AgentLoop directly (unchanged, fast)
- 180k context auto-resend works independently in both Hermes and each BaseSubAgent
